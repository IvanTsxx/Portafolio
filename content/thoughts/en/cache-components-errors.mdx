## Error 1: Accessing runtime APIs inside `'use cache'`

<ErrorCard
  title="Runtime API inside use cache"
  errorMessage="Error: cookies() was called inside a cached function. Runtime APIs cannot be accessed from cached scopes."
  causeLabel="Cause"
  cause="cookies(), headers() and searchParams are runtime APIs: their value depends on the incoming request, it does not exist at build time nor at the moment the cache entry is created. Accessing them inside 'use cache' breaks the guarantee that the function produces the same result for the same inputs."
  fixLabel="Fix"
  fix="Read the value outside the cached scope and pass it as an argument. The argument becomes an automatic part of the cache key, so different values generate different entries."
  tip="If you have a compliance case where you cannot refactor (for example, you need to access cookies inside a third-party library you do not control), there is 'use cache: private' which does allow runtime APIs. Use it as a last resort."
/>

```tsx
// ❌ Breaks at runtime
async function CachedProfile() {
  "use cache";
  const session = (await cookies()).get("session")?.value; // Error
  return <div>{session}</div>;
}
```

```tsx
// ✅ Read outside, pass as argument
async function ProfilePage() {
  const sessionId = (await cookies()).get("session")?.value;
  return <CachedProfile sessionId={sessionId} />;
}

async function CachedProfile({ sessionId }: { sessionId?: string }) {
  "use cache";
  cacheLife("hours");
  // sessionId is part of the cache key → each user has its own entry
  const data = await fetchUserData(sessionId);
  return <div>{data?.name}</div>;
}
```

When you cannot refactor:

```tsx
// ⚠️ Last resort: use cache: private
async function ComplianceComponent() {
  "use cache: private";
  const session = (await cookies()).get("session")?.value; // Allowed
  return <div>{session}</div>;
}
```

## Error 2: Passing non-serializable types as arguments

<ErrorCard
  title="Non-serializable argument"
  errorMessage="Error: Arguments to cached functions must be serializable. Class instances, functions, and Symbols are not supported."
  causeLabel="Cause"
  cause="The cache key is built by serializing the arguments. Types that cannot be serialized (class instances, functions, Symbols, WeakMaps) do not have a stable representation that can be used as a key or transmitted between the server and the client."
  fixLabel="Fix"
  fix="Pass only primitives or plain objects as arguments. If you need data from a complex object, extract only the ID or scalar values and perform the fetch inside the cached function."
/>

```tsx
// ❌ Class instance → not serializable
async function UserProfile({ user }: { user: UserClass }) {
  "use cache";
  return <div>{user.name}</div>;
}

// ❌ Date as argument → also not recommended, inconsistent serialization
async function EventCard({ date }: { date: Date }) {
  "use cache";
  return <div>{date.toISOString()}</div>;
}
```

```tsx
// ✅ Only the ID, fetch inside
async function UserProfile({ userId }: { userId: string }) {
  "use cache";
  cacheLife("days");
  const user = await db.users.findUnique({ where: { id: userId } });
  return <div>{user?.name}</div>;
}

// ✅ String instead of Date
async function EventCard({ dateString }: { dateString: string }) {
  "use cache";
  return <div>{dateString}</div>;
}
```

**Supported argument types:**

| Supported                                          | Not supported               |
| -------------------------------------------------- | --------------------------- |
| `string`, `number`, `boolean`, `null`, `undefined` | Class instances             |
| Plain objects `{ key: value }`                     | Functions                   |
| Arrays                                             | Symbols, WeakMaps, WeakSets |
| `Date`, `Map`, `Set`, `TypedArray`, `ArrayBuffer`  | Instances of `URL`          |
| React elements (pass-through only)                 | Classes with methods        |

## Error 3: Non-deterministic values inside `'use cache'`

<ErrorCard
  title="Non-deterministic value inside use cache"
  errorMessage="Unexpected behavior: Math.random() or Date.now() always return the same value."
  causeLabel="Cause"
  cause="Functions with 'use cache' run only once to produce the cache entry. Any call to Math.random(), Date.now(), or crypto.randomUUID() inside that scope runs on the first render and the result is frozen in the cached entry. All subsequent requests see the same value."
  fixLabel="Fix"
  fix="Move the non-deterministic value outside the cached scope. If you need the value to differ per request, generate it in a dynamic component and pass it as an argument to the cached function."
  tip="Use import { connection } from 'next/server' and await connection() to force a component to run at request-time instead of build time. It is the semantic way to say 'this is dynamic'."
/>

```tsx
// ❌ Runs once, always the same value
async function Banner() {
  "use cache";
  const id = Math.random(); // Always the same number
  const now = Date.now(); // Always the same timestamp
  return <div data-id={id}>Last update: {now}</div>;
}
```

```tsx
// ✅ Generate dynamic value outside, pass as argument
import { connection } from "next/server";

async function BannerWrapper() {
  await connection(); // Forces request-time execution
  const requestId = crypto.randomUUID(); // Different per request
  return <Banner requestId={requestId} />;
}

async function Banner({ requestId }: { requestId: string }) {
  "use cache";
  const bannerData = await fetchBannerData(requestId);
  return <div>{bannerData.text}</div>;
}
```

## Error 4: Dynamic component without `<Suspense>`

<ErrorCard
  title="Missing Suspense boundary around dynamic component"
  errorMessage="Error: A component accessed a runtime API (cookies, headers, searchParams) but is not wrapped in a Suspense boundary. With cacheComponents enabled, dynamic components must be inside Suspense."
  causeLabel="Cause"
  cause="With cacheComponents: true, Next.js builds the static shell at build time. If a component accesses cookies(), headers() or searchParams without being inside Suspense, there is no way to include it in the static shell or to stream it later. The build fails."
  fixLabel="Fix"
  fix="Wrap all components that access runtime APIs in <Suspense> with an appropriate fallback. The shell is served statically and the dynamic component streams when the server resolves it."
/>

```tsx
// ❌ Breaks the build with cacheComponents: true
export default function Page() {
  return (
    <div>
      <StaticNav />
      <UserCart /> {/* accesses cookies() — no Suspense */}
    </div>
  );
}
```

```tsx
// ✅ With Suspense: the shell is static, the dynamic part streams
export default function Page() {
  return (
    <div>
      <StaticNav />
      <Suspense fallback={<CartSkeleton />}>
        <UserCart />
      </Suspense>
    </div>
  );
}

async function UserCart() {
  const userId = (await cookies()).get("userId")?.value;
  const cart = await db.carts.findUnique({ where: { userId } });
  return <CartDisplay items={cart?.items ?? []} />;
}
```

A useful pattern is to create a skeleton component consistent with the real content to avoid layout shifts:

```tsx
function CartSkeleton() {
  return <div className="h-10 w-24 animate-pulse rounded bg-secondary" />;
}
```

## Error 5: `React.cache` does not cross `use cache` boundaries

<ErrorCard
  title="React.cache isolation inside use cache"
  errorMessage="Unexpected behavior: a value stored in React.cache() outside is not visible inside 'use cache'."
  causeLabel="Cause"
  cause="React.cache operates with an isolated scope inside each 'use cache' boundary. Values stored via React.cache() in the parent component are not visible inside a 'use cache' function. Each scope has its own React.cache store."
  fixLabel="Fix"
  fix="Pass data as function arguments instead of using React.cache as a communication channel between scopes. React.cache is still useful for deduplicating requests within the same scope, not for sharing data between scopes."
/>

```tsx
import { cache } from "react";

const store = cache(() => ({ userId: null as string | null }));

function Parent() {
  store().userId = "user_123"; // Stored in parent scope
  return <Child />;
}

async function Child() {
  "use cache";
  const shared = store();
  // ❌ shared.userId is null here
  // 'use cache' has its own React.cache scope
  return <div>{shared.userId}</div>;
}
```

```tsx
// ✅ Pass the value as argument
function Parent() {
  const userId = "user_123";
  return <Child userId={userId} />;
}

async function Child({ userId }: { userId: string }) {
  "use cache";
  const data = await fetchUserData(userId);
  return <div>{data.name}</div>;
}
```

## Error 6: Exporting synchronous functions in a file with `'use cache'` at file level

<ErrorCard
  title="Synchronous export in file-level use cache"
  errorMessage="Error: All exports from a file with 'use cache' at the top level must be async functions."
  causeLabel="Cause"
  cause="When you place 'use cache' at the top of the file, you are telling Next.js that all exports are cacheable functions. Cacheable functions must be async because the result is stored asynchronously."
  fixLabel="Fix"
  fix="Move synchronous utility functions to another file, or change from file-level directive to function-level for the ones you specifically need to cache."
/>

```tsx
// ❌ Breaks: formatDate is synchronous but file has global 'use cache'
"use cache";

export async function getPosts() {
  return db.posts.findMany();
}

export function formatDate(date: Date) {
  // ❌ Error: must be async
  return date.toLocaleDateString();
}
```

```tsx
// ✅ Option 1: move synchronous utilities to another file
// lib/utils/date.ts
export function formatDate(date: Date) {
  return date.toLocaleDateString();
}

// lib/data/posts.ts — without global 'use cache'
export async function getPosts() {
  "use cache";
  cacheLife("days");
  return db.posts.findMany();
}
```

## Quick checklist

Before deploying, check these points:

- Are you reading `cookies()`, `headers()` or `searchParams` outside the `'use cache'` scope and passing them as arguments?
- Are all arguments to your cached functions serializable types (primitives, plain objects)?
- Are all components that access runtime APIs wrapped in `<Suspense>`?
- Are you using `Math.random()` or `Date.now()` inside `'use cache'`? Move them outside.
- Does your file have global `'use cache'` but exports synchronous functions? Refactor.
- Are you expecting a value from `React.cache` to be visible inside a `'use cache'` scope? Use arguments instead.
