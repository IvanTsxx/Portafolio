## Problema

Integrar pagos en aplicaciones modernas suele **degradar arquitecturas bien diseñadas**:

- Lógica duplicada
- Webhooks acoplados a la aplicación
- Pérdida de tipado end-to-end
- Sistema de autenticación fragmentado

## Solución

Este plugin integra pagos, suscripciones y webhooks de **MercadoPago** directamente dentro del ecosistema Better Auth, permitiendo mantener **coherencia arquitectónica** sin sacrificar tipado ni mantenibilidad.

El objetivo es que billing y autenticación convivan de forma **predecible, extensible** y fácil de integrar.

## Principios de diseño

- API completamente **type-safe**
- Separación clara entre **auth y billing**
- Integración simple en **proyectos existentes**
- Arquitectura preparada para **escalar sin fricción**
- Webhooks **desacoplados y tipados**

## Resultado

Menos complejidad operativa, reducción de bugs relacionados con pagos y una integración consistente dentro del flujo de autenticación.

El plugin funciona tanto como **solución productiva** como referencia técnica para integrar sistemas de billing en aplicaciones modernas.
